Dataset Preparation Guide for Machine Learning

1. Data Format Requirements

  • File format should be CSV (Comma Separated Values)
  • First row must contain column headers
  • Each column should represent a feature or the target variable
  • Each row should represent a single observation/sample
Tip: Ensure your CSV file is properly encoded (UTF-8 recommended) to avoid character encoding issues.

2. Data Cleaning Checklist

  • Remove duplicate entries
  • Handle missing values:
    • Remove rows with missing values
    • Or fill missing values with mean/median/mode
    • Or use appropriate imputation methods
  • Remove irrelevant columns (e.g., ID numbers, timestamps if not relevant)
  • Fix inconsistent data (e.g., "Male"/"M", "Female"/"F")
Warning: Removing too many rows with missing values might lead to data loss. Consider the trade-off between data completeness and dataset size.

3. Data Type Requirements

Your dataset should contain only the following types of data:

Data Type Description Example
Numeric Continuous or discrete numbers Age: 25, Price: 199.99
Categorical Finite set of categories Colour: Red/Blue/Green
Binary Two possible values Yes/No, True/False

4. Feature Engineering Guidelines

  • Convert categorical variables to numeric:
    • Use one-hot encoding for nominal categories
    • Use label encoding for ordinal categories
  • Scale numerical features:
    • Standardisation (zero mean, unit variance)
    • Min-Max scaling (scale to specific range)
  • Handle outliers appropriately

5. Dataset Size Considerations

  • Minimum recommended size: 50 samples per feature
  • Balanced classes for classification problems
  • Representative sample of the population
Tip: If your dataset is imbalanced, consider using techniques like:
  • Oversampling (SMOTE)
  • Undersampling
  • Class weights

6. Common Mistakes to Avoid

  • Data leakage (including target-related information in features)
  • Inconsistent data formats (e.g., dates in different formats)
  • Mixed data types in single columns
  • Incorrect handling of categorical variables
  • Not addressing multicollinearity
Important: Always validate your dataset structure and content before training models to avoid errors during the training process.

7. Final Checklist

Task Status
Data is in CSV format
No missing values
Categorical variables encoded
Numerical features scaled
No data leakage
